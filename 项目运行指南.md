# 视障助行多模态项目运行指南

## 1. 项目结构
- 后端云函数代码: [app.py](app.py)
- 前端工程目录: [vision-app](vision-app)
- 前端入口页面: [vision-app/src/App.jsx](vision-app/src/App.jsx)
- 前端状态管理: [vision-app/src/store/useAIStore.js](vision-app/src/store/useAIStore.js)

## 2. 运行前准备
### 2.1 系统环境
- Node.js 18 及以上
- npm 9 及以上
- Python 3.10（仅在本地调试后端时需要）

### 2.2 云服务准备
- 阿里云 OSS Bucket（标准存储-本地冗余）
- OSS CORS 已配置允许跨域上传
- 阿里云函数计算 FC 接口已部署并可访问
- DashScope API Key 已在云函数环境变量中配置

## 3. 前端配置
在 [vision-app/.env](vision-app/.env) 中确认以下变量（按你的真实资源填写）:

VITE_API_URL=你的云函数接口地址/api/analyze
VITE_OSS_REGION=oss-cn-你的地域
VITE_OSS_BUCKET=你的Bucket名称
VITE_OSS_AK=你的AccessKeyId
VITE_OSS_SK=你的AccessKeySecret

# 额度控制建议（试用资源建议保守）
VITE_MAX_ANALYZE_COUNT=25
VITE_REALTIME_INTERVAL_MS=8000
VITE_REALTIME_MAX_FRAMES=8
VITE_VIDEO_KEYFRAME_MAX=10
VITE_VOICE_DIFF_THRESHOLD=0.72

说明:
- VITE_MAX_ANALYZE_COUNT: 单次页面会话最大分析次数
- VITE_REALTIME_INTERVAL_MS: 实时模式抽帧间隔，最小建议 4000
- VITE_REALTIME_MAX_FRAMES: 实时模式单轮最多抓帧数量
- VITE_VIDEO_KEYFRAME_MAX: 视频时长自适应抽帧的上限
- VITE_VOICE_DIFF_THRESHOLD: 语音防抖阈值，越大越不容易播报

## 4. 前端启动
进入前端目录后执行:

npm install
npm run dev

浏览器访问终端输出地址（通常为 http://localhost:5173）。

## 5. 后端运行方式
### 5.1 云端运行（推荐）
- 直接使用 FC 已部署地址作为 VITE_API_URL
- 前端上传到 OSS 后，调用云函数返回文本与音频

### 5.2 本地调试后端（可选）
在项目根目录执行:

pip install flask dashscope
python app.py

本地接口默认地址:
- POST http://127.0.0.1:9000/api/analyze

将前端 VITE_API_URL 临时改为本地地址即可联调。

## 6. 功能使用说明
### 6.1 上传媒体模式
- 支持上传图片
- 支持上传视频（按视频时长自适应抽帧）
- 视频时长越长，抽帧数量越多，但受 VITE_VIDEO_KEYFRAME_MAX 和会话额度上限约束

### 6.2 实时拍摄模式
- 点击打开摄像头
- 点击开始实时识别
- 系统按设定间隔抽帧并分析
- 语音防抖已启用: 新描述与上一条差异不明显时不自动播报

## 7. 快速验证清单
1. 打开页面后能看到 上传媒体 与 实时拍摄 两个标签
2. 上传图片可返回文字描述与语音播放
3. 上传视频可看到按时长抽帧分析提示
4. 实时模式可打开摄像头并周期触发分析
5. 重复场景下会出现语音防抖提示，减少重复播报

## 8. 常见问题排查
### 8.1 OSS 上传失败
- 检查 OSS CORS 是否允许当前来源
- 检查 VITE_OSS_REGION 是否是 oss-cn-xxx 格式
- 检查 AK/SK 与 Bucket 权限

### 8.2 云函数返回 4xx 或 5xx
- 检查 VITE_API_URL 是否正确
- 检查 FC 环境变量中 DashScope Key 是否可用
- 检查 [app.py](app.py) 日志中的错误信息

### 8.3 浏览器不自动播放语音
- 浏览器可能拦截自动播放
- 页面已保留音频控件，可手动点击播放

## 9. 生产与答辩建议
- 试用额度阶段建议使用保守参数:
  - VITE_MAX_ANALYZE_COUNT=20
  - VITE_REALTIME_INTERVAL_MS=10000
  - VITE_REALTIME_MAX_FRAMES=6
  - VITE_VIDEO_KEYFRAME_MAX=8
- 现场演示前先进行一次全链路预热，降低首次调用延迟
